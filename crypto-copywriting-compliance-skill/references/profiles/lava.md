LAVA is a **utility token**. Never imply it's bought for profit. Regulators infer from content + context, not just exact words.

## Banned lexicon (hard fail)

| Verbatim from contract | Same-intent variants also banned |
|---|---|
| investment | investing, invest, invested |
| investor | — |
| purchaser | — |
| profit | profitable, profits |
| gain | gains |
| appreciation (in value) | appreciate, appreciating |
| earn | earnings, earned |
| returns | return on, ROI |
| cash out | cashing out |
| withdraw funds / withdraw cash | withdrawal |
| value (as monetary worth/price) | store of value, value will grow |
| market price | price target, price action |
| speculation | speculate, speculative |

Banned by example (the "what's wrong" tweets):
- "deploying capital" / "deploy capital" — reads as a venture
- "capped supply" / "no inflation" framing — implies holding → appreciation as demand rises
- "buy now," "bull run," "to the moon," "$X a token," "demand will drive…"

### "value" trap
Banned only when it means monetary worth/price. Utility-value ("the value of running a node is reliable RPC access") passes. Price-value ("LAVA's value," "store of value," "value will grow") fails. Check meaning before flagging.

## Approved vocabulary (closed sets — hard fail on stray labels)

People — only:
- **Consumers** — buy subscription plans to access blockchain data
- **Developers** — consume APIs via the Lava Gateway (most important users)
- **Validators** — stake LAVA to secure new blocks
- **Providers** — service relay requests by staking + running RPC nodes (core idea of the network)
- **Users / Token Holders / Members of the Community** — default for anyone not above

No other labels. Never "investors." "node runners" → Validators.

Token — frame only as:
- **Utility** — pays gas on Mainnet; Consumers buy subscriptions (lead with this)
- **Governance** — participate in the DAO
- **Staking & Delegation** — delegate to Validators / stake to run as Provider or Validator
- **Rewards** — awarded for staking, referrals, incentivizing RPC pools

## Channel rules
- **Tweets/threads/captions:** lead with utility; no price, no supply mechanics.
- **Whitepaper / landing / decks:** highest scrutiny — the "public-facing statements" the contract names. No exemption for technical sections.
- **Discord / Telegram / AMA (live):** hardest to undo (screenshots). Same discipline. Boring-but-safe beats clever. Redirect price/airdrop bait to utility.
- **Video:** subtitle/headline/on-screen text count as copy — run them through. No $/rocket B-roll.
